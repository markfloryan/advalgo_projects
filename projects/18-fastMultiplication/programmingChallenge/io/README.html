Explanation Of Test Cases

These test cases are designed to test various different input types, sizes, and potential edge cases. An explanation of each test case is as follows:

Cases 1-6 are handcrafted, and cases 7-20 are computer generated with varying parameters.

## Handcrafted Tests - 1: small nontrivial test case meant as a sample - 2: tests multiple customers, negative coin labels, and inputs which require output labels with absolute value greater than 10000 - 3: tests lots of input labels that are close together with multiple customers - 4: tests a large value of k meant to overflow if you read it as an int - 5: tests for absence of weird behavior in the trivial case (k=1, n=1, only one coin) - 6: tests both a large value of k and input labels far away from each other
